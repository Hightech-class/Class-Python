### [1] 로우레벨(저급) vs 하이레벨(고급) 언어
* **[1-1] 로우레벨 언어**: 컴퓨터 하드웨어 구조 직접 제어, 컴퓨터가 처리하는 방식 그대로 명령 내리는 언어 → **CPU가 직접 실행**
* **[1-2] 하이레벨 언어**: 사람이 이해하기 쉬운 단어 & 문법으로 구성 → **CPU가 직접 실행 X** (중간단계로 컴파일러, 인터프리터가 존재)
    * *예시: C언어, C++, JAVA(객체지향), Python, JavaScript(인간의 언어와 가장 유사, 높은 생산성)*
    * *참고: C언어는 하드웨어 제어와 가독성을 동시에 만족하는 언어로, 상대적으로 로우레벨에 가까움*

---

### [2] 번역 : 컴파일러(Compiler) vs 인터프리터(Interpreter)
* **[2-1] 컴파일러(한꺼번에 기계어로 번역)**
    * 예시: 떡볶이-밀키트 (정해져 있음, 수정X)
    * 장점: 빠른 속도, 높은 보안
    * 단점: 한 줄만 고쳐도 다시 전체 실행 (시간 소요)
* **[2-2] 인터프리터(코드를 한줄씩 읽어가며 즉시 실행)**
    * 예시: 떡볶이-장보기/요리 (시행착오 가능)
    * 장점: 실시간 수정, 쉬운 디버깅
    * 단점: 컴파일러에 비해 실행 속도가 느림
* **[2-3] 하이브리드**: 컴파일러와 인터프리터의 장점 융합
    * 소스코드를 중간 형태(**bytecode**)로 한 번 번역 → 실행 시점에 가상머신(**PVM**)이 처리

### [4] C언어 vs 파이썬 차이
* **예시**: `C언어: int a=10;` / `파이썬: a=10`
* **C언어**: `int b=10;` 지정 시 4byte 공간을 할당하고 변수를 저장. a와 독립적인 메모리 공간을 가짐.
* **파이썬**: `b=10` 지정 시 이미 10이라는 메모리 주소를 가진 것을 a도 알고 b에게도 알려줘서 **메모리 주소를 공유**하게 됨 (임의의 공간 지정 불가).

### [5] 프로그래밍/코딩의 시작
* **핵심 질문**: 어떤 변수를 지정할 것인가? 변수에 어떤 값을 지정할 것인가? 관계를 어떻게 정의할 것인가?
* **변수**: 임의의 데이터 값을 일시적으로 저장하기 위해 사용하는 메모리의 한 영역
* **예시 비교**:
    * **ex1)** 사과 10개, 개당 1000원 → `c = a * b` (관계 지정)
    * **ex2)** 장바구니 과일 a, b, c를 하나씩 꺼내며 합산 → `c = Σ` (누적 합산)
* **Tip**: 코드 작성 시 `*`보다 `+`를 많이 써라 (처리 속도 차이 염두).

### [7] 운영체제(OS)의 역할
* 소프트웨어와 하드웨어 사이를 **중재** (무단 접속 방지, 자원 효율적 배분)
* **시스템 콜(System Call)**: 하드웨어로 가는 유일한 통로
* **추상화 계층**:
    1.  **애플리케이션 계층**: 문제 해결 로직 수행
    2.  **라이브러리/런타임**: 언어별 특수 기능 및 시스템 콜 준비
    3.  **운영체제 계층(커널)**: 자원 배분, 보안, 시스템 콜 처리, 드라이버 제어
    4.  **하드웨어 계층**: 물리적 연산과 저장

### [8] 파이썬 코드 동작 과정
1.  **하이레벨**: 소스코드 → 바이트코드 컴파일
    * 토큰(Token) 분리 → 파싱(Parsing) → 추상구문트리(AST) 생성 → 바이트코드 생성
2.  **미들레벨**: 바이트코드 → 파이썬 가상머신(PVM)
    * **PVM**: 인터프리터 역할, 스택(Stack) 기반 연산, 힙(Heap) 메모리 관리
3.  **로우레벨**: PVM → 하드웨어 실행
    * CPU 데이터 이동, ALU 연산 수행, 전기 신호 출력
